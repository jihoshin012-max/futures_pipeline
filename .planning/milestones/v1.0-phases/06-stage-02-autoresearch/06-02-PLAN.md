**Modify from Stage 04:**
- `parse_program_md()`: Add NEW_FEATURE field parsing. Required fields: metric, keep_rule, budget, new_feature.
- `run_loop()`: Replace engine-specific logic:
  1. Paths: feature_engine_path = autoresearch_dir.parent.parent / "shared/archetypes" / archetype / "feature_engine.py" (or accept via CLI). Actually simpler: the driver copies feature_engine.py from shared/archetypes/{archetype}/ to/from current_best/. Use archetype from program.md or CLI arg.
  2. Pre-loop: verify current_best/feature_engine.py exists (if not, print error and exit — human must seed)
  3. Each iteration: do NOT propose parameters. Instead:
     a. Check if feature_engine.py has changed (compare against current_best/) — if no change, skip (agent hasn't edited yet). For overnight mode: trust that the agent has edited; just run.
     b. Run evaluate_features.py as subprocess: `[sys.executable, str(evaluate_features_path), "--archetype", archetype, "--output", str(result_json_path)]`
     c. Read feature_evaluation.json
     d. Find the feature matching new_feature from program.md in the `features_evaluated` list
     e. **Check entry_time_violation PER-FEATURE:** Read `feature_dict.get("entry_time_violation", False)` from the matching feature dict in features_evaluated. If True, verdict = "entry_time_violation", do NOT keep. Do NOT look for a top-level "violation_count" key — it does not exist in feature_evaluation.json (the dispatcher drops top-level keys).
     f. Otherwise check keep rule: spread > keep_rule AND mwu_p < 0.10
     g. On keep: copy feature_engine.py (from shared/archetypes/{archetype}/) to current_best/feature_engine.py
     h. On revert: copy current_best/feature_engine.py back to shared/archetypes/{archetype}/feature_engine.py
  4. TSV row: stage='02-features', features=new_feature, pf_p1=spread, mwu_p=mwu_p, verdict=kept|reverted|entry_time_violation
