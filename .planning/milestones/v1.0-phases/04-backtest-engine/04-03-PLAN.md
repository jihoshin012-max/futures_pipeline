- Line 1: `# archetype: zone_touch` (per CLAUDE.md convention)
- Define SimResult dataclass (pnl_ticks, win, exit_reason, bars_held)
- def run(bar_df: pd.DataFrame, touch_row: pd.Series, config: dict, bar_offset: int) -> SimResult:
  - Extract mode config from config dict using touch mode
  - Get entry price from touch_row["TouchPrice"]
  - Get direction from touch_row["ApproachDir"] (1 = long, -1 = short)
  - Iterate bars from bar_offset forward:
    - Track MFE (max favorable excursion) in ticks
    - Check trail_steps: if MFE >= trigger_ticks, ratchet stop to max(current_stop, new_stop_ticks) — stop never moves back
    - Check stop: if price adverse >= current stop level, exit stopped_out
    - Check targets: for multi-leg, check each leg_target in order; for single-leg, check the one target
    - Check time_cap_bars: if bars_held >= time_cap_bars, exit at current price
  - Return SimResult

Key implementation details:
- For LONG: MFE = (High - entry) / tick_size in ticks; stop hit when (entry - Low) / tick_size >= stop_ticks
- For SHORT: MFE = (entry - Low) / tick_size in ticks; stop hit when (High - entry) / tick_size >= stop_ticks
- Trail stop ratchets: iterate trail_steps in order, if MFE >= step.trigger_ticks, update current stop to max(current_stop_level, entry +/- step.new_stop_ticks * tick_size * direction)
- Multi-leg: For simplicity in v1, use first leg_target only (single-target mode). Multi-leg partial exits require position tracking — document this as a future enhancement. The per-trade pnl_ticks = first target hit or stop/time_cap exit.
- tick_size: Read from config (the engine passes it through from instruments.md). Default to 0.25 if not present for backward compatibility with tests.

IMPORTANT: Pure function — no file I/O, no prints, no global state. All inputs via function args.

Structure follows RESEARCH.md Pattern 1 exactly:

```python
#!/usr/bin/env python
"""Backtest engine — fixed harness. NEVER MODIFY after Phase 4."""

import argparse, json, sys, importlib
from pathlib import Path
import pandas as pd
import numpy as np

# Add shared modules to path
sys.path.insert(0, str(Path(__file__).resolve().parents[2]))

HOLDOUT_FLAG = Path("stages/04-backtest/p2_holdout/holdout_locked_P2.flag")

def main(config_path, output_path):
    config = load_and_validate_config(config_path)
    check_holdout_flag(config)
    instrument_info = parse_instruments_md(config["instrument"])
    touches, bars = load_data(config["touches_csv"], config["bar_data"])
    adapter = load_scoring_adapter(config["scoring_model_path"], config["archetype"]["scoring_adapter"])
    validate_adapter(adapter)
    touches["score"] = adapter.score(touches)
    route_waterfall(touches, config["routing"])
    simulator = load_simulator(config["archetype"]["simulator_module"], config["archetype"]["name"])
    results = run_simulations(simulator, bars, touches, config, instrument_info)
    metrics = compute_metrics(results, config["active_modes"], instrument_info["cost_ticks"])
    write_result(output_path, metrics)
```

Key functions:
- load_and_validate_config(path): json.load, validate required keys exist, call validate_trail_steps() for each mode's trail_steps
- validate_trail_steps(steps): Implement the 5 validation rules from RESEARCH.md Pattern 4
- check_holdout_flag(config): Check HOLDOUT_FLAG.exists() AND scan touches_csv/bar_data for case-insensitive "p2" in Path.name — per Pitfall 7
- validate_adapter(adapter): Call adapter.score(pd.DataFrame(columns=[...])) in try/except — catch NotImplementedError AND any other exception per Pitfall 6. Use a DataFrame with expected touch columns but zero rows to avoid KeyError before NotImplementedError.
- route_waterfall(touches, routing): Add "mode" column based on score_threshold and seq_limit. Touches above threshold get assigned to first active mode. Touches below threshold or exceeding seq_limit get mode=None (filtered out).
- load_simulator(module_name, archetype_name): sys.path.insert for archetype directory, importlib.import_module, verify run() exists
- run_simulations(): For each touch with mode in active_modes, find bar_offset via searchsorted on datetime, call simulator.run(), collect SimResults
- compute_metrics(results, active_modes, cost_ticks): Calculate pf (with cost), n_trades, win_rate, total_pnl_ticks (net of cost), max_drawdown_ticks, per_mode breakdown
- write_result(path, metrics): json.dump with indent=2

CRITICAL RULES:
- cost_ticks comes from parse_instruments_md() — NEVER hardcode (CLAUDE.md Rule 5)
- PF calculation: sum(winners) / (abs(sum(losers)) + n_trades * cost_ticks) — cost applied at aggregate level
- Sort touches by DateTime before processing (Pitfall 5 determinism)
- Use argparse for --config and --output CLI flags
- The engine file itself is in stages/04-backtest/autoresearch/ — the CLAUDE.md prohibition "NEVER modify backtest_engine.py" applies AFTER this phase completes
