CANDIDATE fields (driver varies): stop_ticks, leg_targets, trail_steps, time_cap_bars
FIXED fields (never change): version, instrument, touches_csv, bar_data, scoring_model_path, archetype, active_modes, routing

Engine CLI: `python backtest_engine.py --config params.json --output result.json`

Result JSON schema:
```json
{"pf": 0.0, "n_trades": 0, "win_rate": 0.0, "total_pnl_ticks": 0.0, "max_drawdown_ticks": 0.0, "per_mode": {"M1": {"pf": 0.0, "n_trades": 0, "win_rate": 0.0}}}
```

Trail step validation rules (driver must enforce before writing params):
1. 0-6 steps allowed
2. trigger_ticks strictly monotonically increasing
3. new_stop_ticks < trigger_ticks for each step
4. new_stop_ticks non-decreasing across steps
5. new_stop_ticks[0] >= 0

results_master.tsv header (24 columns):
run_id, stage, timestamp, hypothesis_name, archetype, version, features, pf_p1, pf_p2, trades_p1, trades_p2, mwu_p, perm_p, pctile, n_prior_tests, verdict, sharpe_p1, max_dd_ticks, avg_winner_ticks, dd_multiple, win_rate, regime_breakdown, api_cost_usd, notes

    **Implementation (GREEN):**
    Create stages/04-backtest/autoresearch/driver.py (~150-200 lines):

    1. `parse_program_md(path)` — read program.md, extract METRIC, KEEP_RULE (float), BUDGET (int) via line-by-line search. Re-read every iteration per Pitfall 2.

    2. `validate_trail_steps(steps)` — enforce all 5 rules from config_schema.md. Return True/False.

    3. `propose_next_params(current_best_config, results_tsv_path)` — random perturbation of CANDIDATE fields around current_best values:
       - stop_ticks: uniform +/- 30 ticks, minimum 10
       - leg_targets: each target +/- 20 ticks, minimum 10, sorted ascending
       - trail_steps: mutate trigger_ticks and new_stop_ticks within valid ranges; validate with validate_trail_steps; retry up to 10 times if invalid
       - time_cap_bars: uniform +/- 20 bars, minimum 10
       Keep FIXED fields unchanged. This is deliberately simple random perturbation — sufficient for loop health verification.

    4. `run_loop(autoresearch_dir, repo_root, max_iterations=None)` — the main loop:
       - Read current_best/exit_params.json as baseline
       - Each iteration: parse_program_md, count results.tsv rows for budget check, propose params, write exit_params.json, subprocess.run engine, read result.json
       - Keep decision: metric > current_best_metric + keep_rule AND n_trades >= 30 (MIN_TRADES constant)
       - If kept: copy exit_params.json -> current_best/exit_params.json, update current_best_metric
       - If reverted: copy current_best/exit_params.json -> exit_params.json
       - If engine non-zero: log EXPERIMENT_ANOMALY to audit_log.md (append), git add audit_log.md, revert params, continue
       - run_id: `git rev-parse --short HEAD` immediately after engine completes
       - Append TSV row with all 24 columns (blank for non-applicable: pf_p2, trades_p2, mwu_p, perm_p, pctile, sharpe_p1, avg_winner_ticks, dd_multiple, regime_breakdown, api_cost_usd)
       - n_prior_tests counted BEFORE experiment (not after — Pitfall 4)
       - Stop when n_prior_tests >= budget OR max_iterations reached

    5. `if __name__ == "__main__"` block: parse --autoresearch-dir (default: script's parent dir) and --repo-root (default: 3 parents up), call run_loop.

    **Create program.md** (<=30 lines, exact template from RESEARCH.md Pattern 2):

    ```
    # Stage 04 Parameter Optimization
    EDIT: exit_params.json only. DO NOT touch backtest_engine.py.
    METRIC: pf
    KEEP RULE: 0.05
    BUDGET: 500

    ## Current search direction
    Random perturbation around current best. Phase 5 validation mode.

    ## Prior best
    [Driver updates from results.tsv automatically]
    ```
