       - Compute H39 `adverse_velocity_ratio`: Ratio of adverse move speed to favorable move speed within current cycle. Track cumulative favorable and adverse bar-second-weighted distances. Return ratio or 0.0 if no favorable movement yet.
       - Return dict `{'adverse_speed': float, 'adverse_velocity_ratio': float}` (0.0 for both when FLAT or no data)

    4. **Add `_get_sim_state(self, bar_idx)` helper method:**
       Return dict with keys TDS expects: `{'direction': self._direction, 'level': self._level, 'anchor': self._anchor, 'position_qty': self._position_qty, 'cycle_start_bar': self._cycle_start_bar, 'bar_idx': bar_idx, 'avg_entry_price': self._avg_entry_price}`. (Note: `self._avg_entry_price` must be tracked -- check if it already exists; if not, compute from trades.)

    5. **Modify simulation loop** (lines 618-638) to follow research Pattern 3:
       - After `close = float(row["Last"])`, compute dynamic features: `dyn_features = self._compute_dynamic_features(bar_idx, row)` and merge with any static features available.
       - Before state machine transitions, if `self._tds is not None`:
         - Call `threat = self._tds.evaluate(row, dyn_features, self._get_sim_state(bar_idx))`
         - Call `action_modifiers = self._tds.apply_response(self._get_sim_state(bar_idx), threat)`
         - Track `self._tds_level_max = max(self._tds_level_max, threat)`
         - If `action_modifiers['force_flatten']` and state is POSITIONED:
           - Call `_finalize_current_cycle_as_tds_exit(bar_idx, dt, close, logger, bars)` (new method -- see below)
           - Continue to next bar (skip state machine)
         - If `self._tds.state.forced_flat` (in cooldown):
           - Decrement `self._tds.state.cooldown_remaining -= 1`
           - If `self._tds.can_reengage(dyn_features)`: set `self._tds.state.forced_flat = False`, allow normal FLAT->SEED
           - Else: `continue` (skip bar)
       - In POSITIONED state: use `effective_step = self._step_dist * action_modifiers.get('step_widen_factor', 1.0)` and `effective_max_levels = self._max_levels - action_modifiers.get('max_levels_reduction', 0)` for distance checks
       - On reversal: call `self._tds.on_reversal()` if TDS active; reset `self._tds_level_max = 0`
       - On add: check `action_modifiers.get('refuse_adds', False)` -- skip add if True; else call `self._tds.on_add()` after add
       - After each bar (end of loop body): call `self._tds.update_cycle_metrics(row, self._get_sim_state(bar_idx))` if TDS active

    6. **Add `_finalize_current_cycle_as_tds_exit()` method:**
       - Similar to the end-of-data finalization (lines 640-657) but with `exit_reason="td_flatten"`
       - Call `logger.finalize_cycle(...)` with `exit_reason="td_flatten"`
       - Reset state: `self._state = "FLAT"`, `self._direction = None`, `self._level = 0`, `self._anchor = None`, `self._position_qty = 0`
       - Increment `self._cycle_id`, reset `self._cycle_trades = []`
       - Set `self._tds.state.forced_flat = True` (enter cooldown)

    7. **Update `TradeLogger.finalize_cycle()`** to use `self._tds_level_max` for `trend_defense_level_max` field (currently hardcoded to 0 at line 297). Pass it as a parameter or read from simulator state.

    8. **Ensure backward compatibility:** When `trend_defense.enabled=false` (or missing from config), the simulation loop must produce EXACTLY identical results to the current code. No TDS objects created, no action modifiers applied. The `_tds is None` guard ensures this.
