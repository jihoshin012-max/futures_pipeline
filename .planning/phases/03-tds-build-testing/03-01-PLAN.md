    - test_detector_consecutive_adds: When consecutive_adds >= threshold AND no qualifying retracement (>= retracement_reset_pct * step_dist in-favor since last add), detector fires; resets on qualifying retracement
    - test_detector_drawdown_budget: When cycle_unrealized_pnl_ticks < -drawdown_budget_ticks, detector fires
    - test_detector_trend_precursor: When precursor_min_signals (default 2) of {price_speed, regime_transition_speed, band_speed_state >= 3, adverse_speed} exceed thresholds, detector fires
    - test_level1_response: When 1 detector fires, apply_response returns step_widen_factor from config (e.g. 1.5), max_levels_reduction=1, refuse_adds=False, force_flatten=False
    - test_level2_response: When 2+ detectors fire, apply_response returns refuse_adds=True, force_flatten=False
    - test_level3_response: When drawdown_budget fires OR 3+ detectors fire, apply_response returns force_flatten=True; cooldown_remaining set to cooldown_bars
    - test_level3_reengage: can_reengage() returns False when cooldown_remaining > 0; returns True when cooldown expired AND evaluate() returns threat < 1

    - test_summary_metrics: get_summary() returns dict with l1_triggers, l2_triggers, l3_triggers matching actual trigger counts after a sequence of evaluate() calls
    - test_on_reversal_resets: on_reversal() resets consecutive_adds=0, clears last_retracement_depths, resets per-cycle state
    - test_on_add_increments: on_add() increments consecutive_adds by 1
