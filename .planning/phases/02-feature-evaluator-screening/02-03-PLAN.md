Apply the Phase 1b cross-bar-type robustness classification framework and produce the ranked advancement list.

<!-- Classification framework from spec Section 3.7 -->
```
Pattern                                          | Classification
All 3 bar types win                              | ROBUST
Vol + tick win, 10-sec fails                     | ACTIVITY_DEPENDENT
One activity-sampled type only                   | SAMPLING_COUPLED
10-sec only wins                                 | TIME_DEPENDENT
All 3 fail                                       | NO_SIGNAL

Advancement decisions:
ROBUST                -> ADVANCE_HIGH
ACTIVITY_DEPENDENT    -> ADVANCE_FLAGGED (requires session-filtered deployment)
TIME_DEPENDENT        -> ADVANCE_FLAGGED (cautious -- time-based filter only)
SAMPLING_COUPLED      -> DO_NOT_ADVANCE
NO_SIGNAL             -> DO_NOT_ADVANCE

H37 special case: only vol + tick evaluated.
  Both win  -> ROBUST_ACTIVITY
  One wins  -> SAMPLING_COUPLED
  Both fail -> NO_SIGNAL

H19 special case: SKIPPED_REFERENCE_REQUIRED (not testable in single-source runner).
  -> Excluded from ranking. Noted for future multi-source testing.
```
