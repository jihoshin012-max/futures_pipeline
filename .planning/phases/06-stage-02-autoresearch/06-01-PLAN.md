  8. For the target feature (read from evaluate()'s optional target_feature param, or evaluate all keys): compute tercile edges on P1a values, bin P1a and P1b, compute spread = mean(best_bin_pnl) - mean(worst_bin_pnl). The feature values determine the bins; the outcome is Reaction (which IS in the original touch_df). This is valid because the feature is evaluated for predictive power of the bin assignment over the outcome. The entry-time guard ensures the FEATURE is computed without lookahead; the OUTCOME (Reaction) is the target variable we're predicting.

  Actually re-reading more carefully: "best-bin vs worst-bin mean trade PnL in ticks" and "Bin edges computed on P1a feature values, applied to P1b touches". The spread is about the FEATURE's predictive power over the outcome. The feature values determine bin assignment; the outcome for each touch is pnl_ticks. For now use Reaction as the pnl proxy.
