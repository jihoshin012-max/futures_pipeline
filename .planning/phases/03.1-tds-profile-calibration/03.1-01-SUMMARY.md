## Decisions Made
- Detector isolation uses extreme "never fire" thresholds: velocity_threshold_sec=0.001, consecutive_adds_threshold=999, drawdown_budget_ticks=999999, precursor_min_signals=99. These values ensure the non-target detectors can never reach their firing conditions.
- Retracement quality detector (D1) cannot be disabled via threshold — it fires based on market data patterns. This is documented behavior; isolating it means all other detectors are disabled, and any L1 response observed is attributable to D1 firing.
- Baseline metrics read directly from profile JSON (not re-run) — Phase 02.1 profiles contain exact no-TDS baseline metrics computed from the same P1a data; re-running would add ~3x compute cost with identical results.
- Experiment 3 degeneration case handled: if best_l1_detector="drawdown_budget", the combined config degenerates to a drawdown-only config at best_l3_threshold, which is equivalent to the Exp 2 winner. This is by design and logged.
