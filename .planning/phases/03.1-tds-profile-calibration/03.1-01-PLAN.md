```python
# Detector 1 (Retracement Quality): fires when last 3 retracement depths are declining
#   - Data-driven, no tunable threshold to disable directly
#   - To "disable": cannot be directly disabled; it depends on market data patterns
#   - When enabled: fires from retracement depth tracking in update_cycle_metrics()

# Detector 2 (Velocity Monitor): fires when (bar_idx - level_zero_bar_idx) < velocity_bars
#   - Controlled by: velocity_threshold_sec -> converted to velocity_bars via median_bar_sec
#   - To disable: set velocity_threshold_sec = 0.001 (converts to velocity_bars=0, never fires since bar_idx > 0)

# Detector 3 (Consecutive Add Counter): fires when consecutive_adds >= consecutive_adds_threshold
#   - Controlled by: consecutive_adds_threshold
#   - To disable: set consecutive_adds_threshold = 999 (never reached)

# Detector 4 (Drawdown Budget): fires when cycle_unrealized_pnl_ticks < -drawdown_budget_ticks
#   - Controlled by: drawdown_budget_ticks
#   - To disable: set drawdown_budget_ticks = 999999 (never reached)
#   - CRITICAL: drawdown_budget_hit ALWAYS triggers Level 3 (force flatten), NOT Level 1

# Detector 5 (Trend Precursor Composite): fires when precursor_signals >= precursor_min_signals
#   - To disable: set precursor_min_signals = 99 (never reached)
#   - In Phase 03.1: only adverse_speed feeds in (H36); H33/H38/H40 not active in baseline profiles

# Level assignment logic:
#   if drawdown_budget_hit OR firing_count >= 3: threat = 3 (emergency)
#   elif firing_count >= 2: threat = 2 (active threat)
#   elif firing_count == 1: threat = 1 (early warning)
#   else: threat = 0
```

    **The 3 experiments are the core abstraction. Each experiment has its own config builder.**

    1. `build_isolated_detector_configs()` -> list of 4 dicts. Each dict is a full TDS config (level_1, level_2, level_3, precursor) where ONE detector is at normal thresholds and ALL others are at extreme "never fire" values:

       Config A (retracement only): All other detectors disabled. NOTE: Detector 1 is data-pattern-driven (fires when last 3 retracement depths are declining). It cannot be "enabled" via a threshold. To isolate it: disable Detectors 2/3/4/5 using extreme thresholds (velocity_threshold_sec=0.001, consecutive_adds_threshold=999, drawdown_budget_ticks=999999, precursor_min_signals=99). Keep L1 response at spec defaults (step_widen_factor=1.5, max_levels_reduction=1).

       Config B (velocity only): velocity_threshold_sec at spec default (60). Disable Detector 3 (consecutive_adds_threshold=999), Detector 4 (drawdown_budget_ticks=999999), Detector 5 (precursor_min_signals=99). NOTE: Cannot fully disable Detector 1 (retracement quality) since it has no tunable threshold. If Detector 1 also fires alongside Detector 2, the run will hit L2 (2 detectors = threat level 2). This is expected and useful data — log it.

       Config C (consecutive_adds only): consecutive_adds_threshold at spec default (3). Disable Detectors 2/4/5. Same caveat about Detector 1.

       Config D (drawdown_budget only): drawdown_budget_ticks at spec default (50). Disable Detectors 2/3/5. CRITICAL: Detector 4 always triggers L3 directly when it fires (see level assignment logic in trend_defense.py line 178: `if drawdown_budget_hit or firing_count >= 3: threat = 3`). This is by design — Experiment 1 Detector 4 tests the L3 force-flatten response, not L1.

       Each config also includes: retracement_reset_pct=0.3 (fixed), cooldown_sec=300 (spec default when L3 active).

    2. `build_drawdown_sweep_configs(thresholds=[30, 40, 50, 60, 80, 100])` -> list of 6 dicts. Each is a TDS config with ONLY Detector 4 active (drawdown_budget_ticks set to the sweep value). Detectors 2/3/5 disabled. Same note about Detector 1 not being fully disableable. cooldown_sec=300 (spec default).

    3. `build_combined_config(best_l1_detector: str, best_l3_threshold: float)` -> dict. Builds a TDS config with the winning L1 detector re-enabled at its spec default threshold AND drawdown_budget_ticks set to best_l3_threshold. Other detectors stay disabled. The `best_l1_detector` param is one of "retracement", "velocity", "consecutive_adds", "drawdown_budget" — though if drawdown_budget won Experiment 1, the combination is just drawdown_budget at the best threshold (Experiment 3 degenerates to a re-run of the Experiment 2 winner).

    4. `build_run_config(profile_data, source_id, tds_config, base_params)` -> dict. Assemble full simulator config from profile martingale params + TDS config dict. Read instrument constants from `_config/instruments.md` via `parse_instruments_md()`. max_contract_size=16 (fixed per Phase 02.1).

    5. `compute_survival_metrics(result, tds, baseline_metrics, bars_df, max_levels, cost_ticks)` -> dict. Compute extended metrics via `compute_extended_metrics()`, get TDS summary via `tds.get_summary()`, compute deltas: worst_dd_reduction = baseline["worst_cycle_dd"] - extended["worst_cycle_dd"], max_level_pct_reduction = baseline["max_level_exposure_pct"] - extended["max_level_exposure_pct"], pnl_impact_ticks = extended["total_pnl_ticks"] - baseline["total_pnl_ticks"], tail_ratio_delta = extended["tail_ratio"] - baseline["tail_ratio"]. Count td_flatten cycles from result.cycles["exit_reason"]. Compute l3_recovery_bars_avg from td_flatten cycle durations.

    6. `load_profiles()` -> dict. Load all 3 profile JSONs from `profiles/` directory.

    7. `run_experiment(experiment_num, args)` -> pd.DataFrame. Main runner: for each profile, for each bar_type in profile, load bars, run no-TDS baseline to get fresh baseline_metrics (do NOT trust profile JSON metrics — they were computed with potentially different handling), then for each config in the experiment's config list: build full config, run RotationalSimulator, verify sim._tds is not None, compute survival metrics, append row. Return DataFrame with results.
