## Decisions Made

- **L3 over-trigger root cause**: The drawdown_budget force-flatten at 30-100 tick thresholds fires on virtually every cycle in P1a data. NQ moves 30-100 ticks per cycle routinely; the threshold range must be much higher (>500 ticks) or the mechanism is wrong for these profile configs. This is a calibration finding, not a bug.
- **Selection fallback for guard failures**: For 8/9 combinations where all configs failed the guard, select_best_configs() picks the least-bad by pnl_impact_ticks and sets `_no_guard_pass=True`. Phase 04 must treat these as "TDS not beneficial" for these bar types/profiles.
- **--auto-exp3 preferred over manual --best-l1/--best-l3**: Per-combination winners differ; the flag approach avoids 9 separate manual CLI invocations.

## Issues Encountered
