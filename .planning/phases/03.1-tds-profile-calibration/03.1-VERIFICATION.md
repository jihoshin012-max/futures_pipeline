## Gaps Summary

One gap blocks a clean pass: the `test_output_json_schema` test at line 742 of `test_tds_calibration.py` asserts `entry["trend_defense"]["enabled"] is True` for all 9 entries. This assertion was written during Plan 01 (before experiments ran) under the assumption that every calibrated entry would have TDS active. The experiments revealed that TDS is beneficial for only 2/9 combinations (MAX_PROFIT/10sec and MOST_CONSISTENT/10sec). The human-approved disposition for the other 7 combinations is TDS disabled (enabled=False). The production code and JSON output are correct — the test is stale.

The fix is a one-line change: replace the hard assertion with a check that `"enabled"` exists and is a boolean, or restructure to assert enabled=True only for 10sec bar types and enabled=False for vol/tick bar types.

The underlying phase goal — calibrated TDS configs per profile per bar type in best_tds_configs.json for Phase 04 consumption — is fully achieved. All 99 experiment runs completed, all 4 TDS-CAL requirements satisfied, human checkpoint approved.
