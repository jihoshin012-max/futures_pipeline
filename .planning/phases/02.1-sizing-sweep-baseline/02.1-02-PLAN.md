**Part A: compute_extended_metrics() in rotational_engine.py**

Add a new function `compute_extended_metrics(cycles: pd.DataFrame, cost_ticks: float, bars_df: pd.DataFrame, max_levels: int) -> dict` BELOW the existing `compute_cycle_metrics()`. This function calls `compute_cycle_metrics()` first to get base metrics, then adds:

1. `worst_cycle_dd`: `abs(min(cycles["net_pnl_ticks"]))` — worst single-cycle loss (NOT cumulative drawdown)
2. `max_level_exposure_pct`: `(cycles["max_level_reached"] == max_levels).sum() / len(cycles) * 100` — % of cycles that hit the max level cap
3. `tail_ratio`: `avg_winner_ticks / abs(avg_loser_ticks)` — from base metrics, guard divide-by-zero
4. `calmar_ratio`: `total_pnl_ticks / max_drawdown_ticks` — from base metrics, guard divide-by-zero
5. `sortino_ratio`: `np.mean(pnl) / np.std(pnl[pnl < 0])` — guard std==0
6. `winning_session_pct`: group cycles by trading date, compute per-date net PnL, % of dates with positive total. Map date via `bars_df.iloc[cycle_start_bar]["datetime"].date()` — needs the `start_bar` column from cycles. If `start_bar` not in cycles columns, use cycle index as fallback and compute from the first bar of each cycle. NOTE: Check cycles DataFrame for a column containing start bar index — the TradeLogger records it.
7. `max_dd_duration_bars`: from cumulative PnL series, find the longest stretch between a peak and recovery to that peak level. If no recovery, duration extends to end of series.

Return merged dict: {**base_metrics, **extended_metrics}.

Handle edge cases: empty cycles DataFrame returns zeros for all extended metrics. Single cycle returns that cycle's values.

   - Return list of unique (step_dist, max_levels, max_total_position) tuples.
   - Print dedup stats: "Deduplicated {raw} -> {unique} combinations per bar type"

3. **Config construction** per combo: deep-copy template, set:
   - `cfg["hypothesis"]["trigger_params"]["step_dist"] = sd`

   - Per-source config: `cfg["bar_data_primary"] = {source_id: path}` (for RTH filter)
