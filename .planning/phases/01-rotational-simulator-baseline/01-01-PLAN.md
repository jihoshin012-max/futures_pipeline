Create `shared/archetypes/rotational/rotational_simulator.py` with line 1: `# archetype: rotational`

**Classes to implement:**

1. **SimulationResult** (dataclass):
   - trades: pd.DataFrame (columns: bar_idx, datetime, action, direction, qty, price, level, anchor, cost_ticks, cycle_id)
   - cycles: pd.DataFrame (all fields from spec Section 6.4 — see cycle record fields in context)
   - bars_processed: int

2. **FeatureComputer**:
   - `__init__(self, config)` — for baseline, this is minimal
   - `compute_static_features(self, bar_df)` — returns bar_df with any additional columns. For baseline, the CSV already has ATR and SD bands, so this is essentially a no-op pass-through. Structure it so Phase C can add computed features.

3. **TradeLogger**:
   - `__init__(self, tick_size, cost_ticks)` — store instrument constants
   - `log_action(self, bar_idx, datetime, action, direction, qty, price, level, anchor, cycle_id)` — append to trades list
   - `finalize_cycle(self, cycle_id, start_bar, end_bar, direction, trades_in_cycle, bars_df)` — compute cycle record with all Section 6.4 fields:
     - avg_entry_price: weighted average of all entry prices in cycle
     - gross_pnl_ticks: (exit_price - avg_entry_price) / tick_size * total_qty (flip sign for Short)
     - net_pnl_ticks: gross_pnl - sum of (cost_ticks * qty) for each action in cycle
     - max_adverse_excursion_ticks: worst bar-level unrealized PnL during cycle (in ticks)
     - max_favorable_excursion_ticks: best bar-level unrealized PnL during cycle (in ticks)
     - retracement_depths: list of retracement ratios when price pulls back from MFE
     - time_at_max_level_bars: count of bars spent at highest level reached
     - trend_defense_level_max: 0 for baseline (no TDS)
     - exit_reason: "reversal" for baseline
   - `get_trades_df(self)` -> pd.DataFrame
   - `get_cycles_df(self)` -> pd.DataFrame

4. **RotationalSimulator**:
   - `__init__(self, config, bar_data, reference_data=None)`:
     - Extract params: step_dist from config.hypothesis.trigger_params.step_dist
     - Extract martingale: initial_qty, max_levels, max_contract_size from config.martingale
     - Extract instrument: tick_size, cost_ticks from config._instrument
     - Extract period from config.period
     - Store bar_data (DataFrame from load_bars)
     - Initialize state: state="FLAT", direction=None, level=0, anchor=None, position_qty=0, cycle_id=0

   - `_filter_bars(self, bars)` -> pd.DataFrame:
     - **Date filtering**: If period is "P1a" or "P1b", compute midpoint of P1 (2025-09-21 to 2025-12-14). P1a = start..midpoint, P1b = midpoint..end. Filter bars by datetime column. For P1 or P2, no date filter needed (full file).
     - **RTH filtering**: If source_id contains "10sec", filter bars where Time is >= "09:30:00" and < "16:00:00". Parse Time column as time for comparison. Do NOT filter vol/tick bars (they are inherently session-filtered by their construction).
     - Return filtered, reset-index DataFrame.

   - `run(self)` -> SimulationResult:
     - Apply _filter_bars
     - Initialize FeatureComputer, compute static features
     - Initialize TradeLogger
     - Process every bar sequentially:
       - Get close price from "Last" column (CSV header calls it Last, not Close)
       - If state == FLAT: execute SEED (buy InitialQty Long)
       - If state == POSITIONED:
         - Calculate distance = close - anchor (signed)
         - For Long: in_favor = distance >= step_dist, against = -distance >= step_dist
         - For Short: in_favor = -distance >= step_dist, against = distance >= step_dist
         - If in_favor: execute REVERSAL
         - Elif against: execute ADD
     - After all bars: finalize any open cycle (exit_reason="end_of_data", use last bar's close)
     - Return SimulationResult(trades=logger.get_trades_df(), cycles=logger.get_cycles_df(), bars_processed=len(filtered_bars))

   **SEED action:**
   - direction = "Long", qty = initial_qty, anchor = close, level = 0, state = "POSITIONED"
   - cycle_id increments
   - Log action: (bar_idx, datetime, "SEED", "Long", initial_qty, close, 0, close, cycle_id)

   **REVERSAL action:**
   - Finalize current cycle (exit at current close)
   - Flatten: log "FLATTEN" action with current position_qty
   - Enter opposite: new_direction = "Short" if was "Long" else "Long"
   - qty = initial_qty, anchor = close, level = 0
   - cycle_id increments
   - Log "REVERSAL" action
   - state remains "POSITIONED"

   **ADD action:**
   - add_qty = initial_qty * (2 ** level)
   - If add_qty > max_contract_size: add_qty = initial_qty, level = 0 (cap reset)
   - Else: level += 1
   - anchor = close
   - position_qty += add_qty
   - Log "ADD" action

**RTH filter specifics:**
- The Time column in bar data is a string like "09:30:00" or "9:30:00"
- Parse and compare as time objects
- Filter BEFORE simulation (not during)

**Date filter specifics:**
- P1 rotational: 2025-09-21 to 2025-12-14
- Midpoint: compute as start + (end - start) / 2
- P1a: start <= datetime.date() <= midpoint
- P1b: midpoint < datetime.date() <= end
- Use the datetime column (already parsed by load_bars)

**Critical details:**
- "Last" column = Close price (CSV header convention)
- tick_size = 0.25 for NQ (from instruments.md, passed via config._instrument)
- cost_ticks = 3 for NQ (from instruments.md, passed via config._instrument)
- Reversal cost = cost_ticks * flatten_qty + cost_ticks * initial_qty (two actions)
- All PnL in ticks (divide price diff by tick_size)
- retracement_depths stored as JSON-serializable list in cycle DataFrame
