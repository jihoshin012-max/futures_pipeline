**Script design:**
- Import RotationalSimulator directly (not through engine) for efficiency — avoids re-loading/re-validating config 33 times.

- Load bar data ONCE per bar type, then reuse across all StepDist values.

**Implementation:**

```python
def run_sweep(config_template, bar_data_dict, instrument_info, step_values):
    """Run parameter sweep across StepDist values for all bar types.

    Args:
        config_template: Base config dict (modified per iteration)
        bar_data_dict: {source_id: DataFrame} - pre-loaded bar data
        instrument_info: {tick_size, tick_value, cost_ticks}
        step_values: list of StepDist values to test

    Returns:
        dict: {step_dist: {source_id: metrics_dict}}
    """
```

- For each step_dist in [1.0, 1.5, 2.0, 2.5, 3.0, 3.5, 4.0, 4.5, 5.0, 5.5, 6.0]:
  - For each (source_id, bars) in bar_data_dict:
    - Deep-copy config_template, set hypothesis.trigger_params.step_dist = step_dist
    - Create RotationalSimulator(config, bars), call run()
    - Compute metrics via compute_cycle_metrics(result.cycles, cost_ticks)
    - Add bars_processed to metrics
    - Store in results[step_dist][source_id]

**CLI entry point:**
```
python run_sweep.py [--output-dir shared/archetypes/rotational/baseline_results/]
```
